## How to Contribute

We welcome contributions in the form of bug fixes, feature enhancements, or documentation improvements. Please follow the steps below to contribute to the project.

### Fork the Repository

Click the "Fork" button at the top right corner of the repository page on GitHub. This creates your own copy of the repository.

### Clone Your Fork

Clone the forked repository to your local machine:

```bash
git clone https://github.com/SE-Fall24-G73/git-homework.git
```


### Create a Branch

Create a new branch to work on your changes:

```bash
git checkout -b feature/your-feature-name
```


### Make Your Changes

Make the necessary changes in your local repository. Ensure that your changes align with the project's code style and guidelines.

### Commit Your Changes

Stage the files you have changed and commit your changes with a descriptive message:

```bash
git add .
git commit -m "Add feature: description of the feature"
```

*Follow the [Conventional Commits](https://www.conventionalcommits.org/) standard for commit messages if possible.*

### Push to GitHub

Push your branch to your forked repository:

```bash
git push origin feature/your-feature-name
```

### Submit a Pull Request

1. Go to your forked repository on GitHub.
2. Click the "Compare & pull request" button next to your recently pushed branch.
3. Fill out the pull request template, providing a clear description of your changes.
4. Submit the pull request.

Your pull request will be reviewed by the maintainers. Please be patient, as it may take some time to get feedback.

## Code Style Guidelines

To maintain consistency in the codebase, please adhere to the following guidelines:

- **HTML/CSS:**
  - Use consistent indentation (2 or 4 spaces).
  - Write semantic HTML tags.
  - Use lowercase for all HTML tags and attributes.
  - Comment your code where necessary.

- **Commit Messages:**
  - Use the imperative mood (e.g., "Add feature," not "Added feature").
  - Be concise but descriptive.
  - Reference any relevant issues (e.g., "Fixes #12").

- **File Naming:**
  - Use lowercase letters and hyphens for file names (e.g., `index.html`, `team-styles.css`).

## Reporting Issues

If you find bugs, have feature requests, or encounter any issues, please open an issue on GitHub:

1. Navigate to the [Issues](https://github.com/SE-Fall24-G73/git-homework/issues) tab.
2. Click on "New Issue."
3. Provide a clear and descriptive title and detailed information about the issue.
